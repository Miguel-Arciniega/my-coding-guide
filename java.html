Java ships every 6 months, but companies standardize on the Long-Term Support releases. Those are your four exam topics. Learn the one flagship idea per release first; the details hang off it.

Why these four? They're the LTS releases — patched for years, so production targets them. The in-between releases (9, 10, 12–16, 18–20) are where features incubate as previews; an LTS is where they become standard. Knowing "which LTS made it final" is the senior-sounding detail.

2014 · LTS · the big one

Java 8 — functional Java

The release that redefined idiomatic Java. If you internalize one version cold, make it this — every later feature assumes you write streams and lambdas fluently.

Memory bucket
LambdasStreamsOptionaljava.timedefault methods

What landed

FeatureWhat it doesWhy it matters
Lambdas(a, b) -> a + b — pass behavior as a value.Kills anonymous-class boilerplate; the foundation everything else builds on.
Functional interfacesOne abstract method: Function, Predicate, Consumer, Supplier. @FunctionalInterface enforces it.The types a lambda fits into. Know the four core shapes.
Stream APIfilter / map / reduce / collect over collections — lazy, chainable, parallelizable.Declarative data processing. The single most-tested Java 8 topic.
Method referencesUser::getName — shorthand for a lambda that just calls one method.Cleaner than u -> u.getName() when you're not transforming.
Optional<T>A box that holds a value or nothing — map / filter / orElseThrow.Encodes "might be absent" in the type instead of returning null.
Default methodsInterfaces can ship method bodies: default boolean isEmpty(){…}.Lets interfaces evolve without breaking implementers (how Collection.stream() was added).
java.timeLocalDate, LocalDateTime, Instant, Duration — immutable, thread-safe.Replaces the broken, mutable Date/Calendar. Always reach for java.time.
CompletableFutureComposable async pipelines: thenApply, thenCompose, allOf.Non-blocking orchestration without callback hell.

Stream mental model: a source (collection) → zero or more intermediate ops (filter, map, lazy) → one terminal op (collect, reduce, forEach) that triggers execution. Nothing runs until the terminal op. Collectors.groupingBy is the one to know — it's SQL GROUP BY for objects.

🎤 Say it out loud

"Java 8 made Java functional — lambdas let you pass behavior, the Stream API processes collections declaratively, Optional models absence in the type system, and java.time finally fixed dates. Default methods are the quiet hero: they're how the JDK added stream() to Collection without breaking every implementation."

i
Pre-8 baseline (assume these too): generics (Java 5), enhanced for-loop & autoboxing (5), try-with-resources & the diamond operator <> (7). Interviewers treat these as table stakes, not "new."

2018 · LTS · the migration target

Java 11 — the modern baseline

The first LTS after 8, so it's the version most teams actually upgraded to. Fewer headline language features, more "the platform grew up": a real HTTP client, handy String/File helpers, and a lighter runtime.

Memory bucket
var in lambdasString helpersFiles.readStringHttpClientrun .java directly

What landed

FeatureWhat it doesWhy it matters
var in lambdas(var x, var y) -> x + y — local-variable syntax inside lambda params.Lets you put annotations on inferred params. (Local var itself arrived in Java 10.)
String methodsisBlank(), strip(), lines(), repeat(n).Everyday helpers you used to pull from Apache Commons. strip() is Unicode-aware, unlike trim().
Files read/writeFiles.readString(path) / Files.writeString(path, s).One-liners for whole-file text I/O — no reader/writer boilerplate.
HttpClientStandard java.net.http client — sync + async, HTTP/2, WebSocket.Replaces clunky HttpURLConnection; no more reaching for a third-party client for simple calls.
Run source directlyjava Hello.java — no separate javac step.Scripting-style quick runs and teaching; single-file programs just work.
Collection/PredicateCollection.toArray(String[]::new), Predicate.not(...).Small ergonomics — not() reads better than negating inside the lambda.

🎤 Say it out loud

"Java 11 was the LTS everyone migrated to after 8. It's lighter on language features but practical: a built-in HttpClient with HTTP/2, String helpers like isBlank and strip, Files.readString, and you can run a .java file without compiling first. The big context is Java 9's module system and the removal of the bundled Java EE modules."

i
Lineage between 8 and 11: Java 9 added the module system (module-info.java), the jshell REPL, and collection factories (List.of, Map.of); Java 10 added local-variable var. Both are pre-11 but commonly attributed to "modern Java."

2021 · LTS · the language modernizes

Java 17 — modern language features go final

The most interview-relevant jump after 8. Features that had been previewing for years all became standard here: records, sealed types, pattern matching for instanceof, switch expressions, and text blocks.

Memory bucket
recordssealedpattern instanceofswitch expressionstext blocks

What landed (final in 17, after previews)

FeatureWhat it doesWhy it matters
Recordsrecord Point(int x, int y){} — immutable data carrier; auto constructor, accessors, equals/hashCode/toString.Kills DTO/value-object boilerplate. The single most-used new feature.
Sealed classessealed interface Shape permits Circle, Square — restricts who can extend/implement.Closed hierarchies the compiler can reason about → exhaustive switches.
Pattern matching instanceofif (o instanceof String s) — test and bind in one step.Removes the redundant cast after every instanceof check.
Switch expressionsvar r = switch(day){ case MON -> 1; … }; — arrow labels, returns a value, yield for blocks.No fall-through bugs, no missing break, exhaustive on enums.
Text blocksTriple-quoted multi-line strings: """…""".Readable JSON/SQL/HTML literals without \n and + soup.
Helpful NPEsNullPointerException names which variable was null."Cannot invoke … because order.customer is null" — instant debugging.

🎤 Say it out loud

"Java 17 is where the language modernized — records for immutable data, sealed types for closed hierarchies, pattern matching for instanceof, switch expressions, and text blocks. These had all been previewing since 14–16; 17 made them standard. Together they cut a ton of boilerplate and set up exhaustive pattern matching in 21."

2023 · LTS · the current target

Java 21 — virtual threads & complete pattern matching

The newest LTS and today's "modern Java." Two themes: virtual threads make ordinary blocking code scale to millions of tasks, and pattern matching finishes the job started in 17.

Memory bucket
virtual threadspattern matching switchrecord patternssequenced collections

What landed

FeatureWhat it doesWhy it matters
Virtual threadsLightweight threads (Project Loom): Thread.ofVirtual(), newVirtualThreadPerTaskExecutor(). Millions are cheap.Write simple blocking code; the JVM unmounts a virtual thread from its carrier while it waits. Scales like async without the callback model.
Pattern matching switchswitch(shape){ case Circle c -> …; case Square s -> … } — type patterns + guards (when).With sealed types the switch is exhaustive — the compiler enforces every case. Replaces the visitor pattern.
Record patternsDestructure in place: case Point(int x, int y) -> …, nestable.Pull fields straight out of records inside a switch/instanceof — no accessor calls.
Sequenced collectionsSequencedCollection with getFirst(), getLast(), reversed().A uniform first/last API across List, Deque, LinkedHashSet.

The payoff: a virtual thread costs ~KBs, not ~MBs. While it blocks on I/O the JVM parks it and reuses the OS carrier thread, so a few OS threads run millions of tasks. You keep the simple thread-per-request model and still scale.

Sealed + record patterns → exhaustive switch

17's sealed records, destructured exhaustively in 21

🎤 Say it out loud

"Java 21's headline is virtual threads — you write plain blocking code and it scales to millions of concurrent tasks because the JVM unmounts a virtual thread while it waits on I/O. It also completes pattern matching: pattern matching for switch plus record patterns, which combined with 17's sealed types give you compiler-checked exhaustive matching. Sequenced collections round it out with a uniform getFirst/getLast."

i
Still preview in 21 (don't claim as final): string templates, structured concurrency, and scoped values were preview features in 21 — name them as "coming," not done.
